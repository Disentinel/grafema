# REG-67: Release workflow — ветка stable и семантическое версионирование

**Labels:** v0.3, Feature
**Linear:** https://linear.app/reginaflow/issue/REG-67/release-workflow-vetka-stable-i-semanticheskoe-versionirovanie

## Цель

Создать процесс релизов для Navi: стабильная ветка, версионирование, возможность использовать рабочую версию инструмента независимо от состояния main.

## Зачем

1. **Хуки для Claude Code** — нужна рабочая версия Navi для проверки импортов, даже когда main сломан
2. **Dogfooding** — использовать Navi для разработки Navi
3. **Предсказуемость** — понятно какая версия где используется

## Что нужно сделать

### 1. Ветка stable
- Создать ветку stable от текущего main
- stable обновляется только при релизах

### 2. Семантическое версионирование
- Использовать semver для пакетов
- Теги: v0.1.0, v0.2.0-beta.1, etc.

### 3. Релизный процесс
- Обновить CHANGELOG.md
- Bump версии во всех package.json
- Создать тег
- Merge в stable
- Опубликовать в npm (когда готовы)

### 4. Документация
- Описать процесс в CONTRIBUTING.md или отдельном RELEASING.md
