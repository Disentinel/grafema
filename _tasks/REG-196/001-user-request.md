# REG-196: RFDB Performance Benchmarks + Regression Tests

## Цель

Убедиться что RFDB становится быстрее от версии к версии, а не медленнее. Автоматически ловить регрессии производительности.

## План

1. **Benchmark suite**
   * Набор типовых операций: вставка, запросы, joins, агрегации
   * Синтетические датасеты разного размера (100, 1K, 10K, 100K записей)
   * Воспроизводимые условия (фиксированный seed, cold start)
2. **Baseline + tracking**
   * Зафиксировать текущие метрики как baseline
   * Хранить историю результатов (JSON в репо или отдельный артефакт)
3. **CI интеграция**
   * Запуск бенчмарков на PR (опционально, долго)
   * Алерт если деградация > X% от baseline
4. **Отчётность**
   * Вывод в консоль: ops/sec, latency p50/p95/p99
   * Сравнение с предыдущим релизом

## Критерии готовности

- [ ] Benchmark suite покрывает основные операции RFDB
- [ ] Baseline зафиксирован для текущей версии
- [ ] CI ловит регрессии > 10%
- [ ] Документация как запускать локально
